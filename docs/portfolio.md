# Описание проекта для портфолио / резюме

## Краткое описание (1–2 предложения)

**ecommerce-api-fastapi** — production-ready REST API бэкенда интернет-магазина
на FastAPI и PostgreSQL: авторизация по JWT с ролями, каталог товаров с
фильтрами, корзина, оформление заказов со списанием остатков и система отзывов.
Полностью контейнеризован (Docker Compose), покрыт автотестами и CI.

## Что демонстрирует проект (для собеседования)

- **Чистая архитектура**: разделение на слои `routers / schemas / models / core`,
  переиспользуемые зависимости для аутентификации и проверки ролей.
- **Безопасность**: JWT access-токены, bcrypt-хеширование паролей,
  ролевое разграничение доступа (RBAC) на уровне эндпоинтов.
- **Работа с БД**: SQLAlchemy 2.0 (typed `Mapped`), связи один-ко-многим и
  many-to-many через ассоциативные таблицы, каскадные удаления, уникальные
  ограничения (один отзыв на пару пользователь–товар).
- **Бизнес-логика**: оформление заказа из корзины как транзакция —
  проверка остатков, фиксация цены на момент покупки, списание склада,
  очистка корзины.
- **Качество**: 16 автотестов на pytest с изолированной БД, отчёт о покрытии,
  CI на GitHub Actions (миграции на PostgreSQL + тесты).
- **DevOps**: многослойный Dockerfile, docker-compose с healthcheck,
  entrypoint с ожиданием БД, автоприменением миграций и сидированием.
- **DX**: Swagger/OpenAPI из коробки, seed-данные для мгновенной демонстрации,
  подробный README на русском.

## Ключевые технические решения

| Решение | Обоснование |
|---------|-------------|
| FastAPI + Pydantic v2 | Асинхронный фреймворк с автогенерацией OpenAPI и строгой валидацией |
| SQLAlchemy 2.0 (`Mapped`) | Типобезопасные модели, современный декларативный стиль |
| JWT вместо сессий | Stateless-аутентификация, удобна для SPA/мобильных клиентов |
| Фиксация цены в `order_items` | Цена товара может измениться — заказ хранит историческую цену |
| SQLite в тестах | Быстрый изолированный прогон без внешних зависимостей в CI |
